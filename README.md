## 1. 리눅스란?
<img width="445" height="220" alt="Linux" src="https://github.com/user-attachments/assets/e6b8edf6-f70a-47ee-9d2a-5f336cf41870" />

**리눅스(Linux)** 는 유닉스 철학을 따르는 **오픈소스 운영체제 커널**이며, 일반적으로 커널(Linux)과 GNU 유틸리티, 패키지 관리자 등을 묶은 **배포판(예: Ubuntu/Debian, Fedora/RHEL, Arch)** 형태로 사용합니다.  
안정성·보안·유연성 덕분에 서버/클라우드, 개발용 워크스테이션, 임베디드/IoT(안드로이드도 리눅스 커널 기반)까지 폭넓게 쓰이며 **현업 표준**으로 자리 잡았습니다.

이 문서의 아래부터는 리눅스 환경에서 **프로세스와 작업을 다룰 때 가장 자주 쓰는 네 가지 명령어**인 `top`, `ps`, `jobs`, `kill`을 자연스럽게 이어서 정리합니다.

## 2. 명령어별 정리

### 2.1 `top`
시스템 부하, CPU/메모리 사용률, 프로세스 상위 소비자 등을 **실시간**으로 보여준다.

**주요 옵션/키**
- 실행 후 키보드 단축키:
  - `P`: CPU 사용률로 정렬
  - `M`: 메모리 사용률로 정렬
  - `1`: 각 CPU 코어 사용률 표시
  - `k`: 선택한 PID에 신호 보내기(기본 `TERM`)
  - `q`: 종료
- 시작 옵션:
  - `-o %MEM` : 메모리 기준 정렬
  - `-p <PID>` : 특정 PID만 보기

**예시**
```bash
top                   # 기본 실행
top -o %MEM           # 메모리 사용량 기준 정렬로 시작
```

### 2.2 `ps`
현재 프로세스의 스냅샷을 출력한다. 파이프/grep과 조합해 많이 사용.

**자주 쓰는 조합**
- ps aux : 모든 사용자/터미널의 프로세스 전체 목록
- ps -ef : 표준(전통적) 포맷의 전체 목록
- ps aux | grep <이름> : 특정 이름 검색
- ps -o pid,ppid,cmd -p <PID> : 특정 PID 상세 컬럼 지정

**예시**
```bash
ps aux | head
ps aux | grep nginx | grep -v grep
ps -o pid,ppid,%cpu,%mem,cmd -p 12345
```

### 2.3 `jobs`
현재 셸 세션에서 실행한 백그라운드/중지된 작업 목록을 보여준다. (&, Ctrl+Z, bg/fg와 함께 사용)

**핵심 사용법**
- jobs : 현재 셸의 잡 목록
- jobs -l : PID까지 함께 표시
- bg %1 / fg %1 : 잡 번호로 백그라운드/포어그라운드 전환

**예시**
```bash
sleep 1000 &   # 백그라운드 잡 생성
jobs -l        # 잡 번호(%)와 PID 확인
fg %1          # 1번 잡을 포어그라운드로 가져오기
```

### 2.4 `kill`
프로세스(PID)나 잡 번호(%)에 **신호(signal)**를 보낸다. 종료가 일반적이지만 다양한 신호 가능.

**대표 신호**
- TERM (15): 정상 종료 요청(기본) → 우선 권장
- KILL (9): 강제 종료(마지막 수단)
- HUP (1): 설정 재적용(데몬 재시작 용도로 쓰이는 경우 多)
- STOP / -CONT: 일시중지 / 재개

**예시**
```bash
kill -TERM <PID>      # 정상 종료
kill -9 <PID>         # 강제 종료(주의)
kill -l               # 사용 가능한 신호 목록 출력
kill %1               # jobs의 1번 잡에 신호 보내기 (현재 셸 컨텍스트)
```

### 3. 요약 표

- *`top`*: 실시간 프로세스/시스템 자원 모니터링
- *`ps`*: 현재(또는 특정 조건의) 프로세스 목록을 스냅샷으로 조회
- *`jobs`*: 현재 셸 세션에서 백그라운드/중지된 Job 상태 확인
- *`kill`*: PID나 Job 번호로 프로세스/잡에 신호(signal) 전송(종료 등)


> **Tip**  
> 실시간 상태는 `top`, 정적인 목록은 `ps`, 현재 셸의 백그라운드 작업은 `jobs`, 종료/신호 전송은 `kill`을 사용한다.


| 명령  | 목적               | 대표 상황                | 핵심 사용법/키                                 | 기억 포인트                        |
|------|--------------------|--------------------------|-----------------------------------------------|------------------------------------|
| `top` | 실시간 모니터링     | 갑작스러운 부하 원인 추적 | `P`, `M`, `1`, `k`, `u`, `q` / `-o %MEM`, `-p` | 화면에서 정렬/필터/신호까지 바로 가능 |
| `ps`  | 스냅샷 목록         | 조건 검색·스크립트 처리   | `aux`, `-ef`, `-o`, `--sort`                   | `grep/awk/sort` 파이프 조합이 핵심   |
| `jobs`| 현재 셸 잡 관리     | `&`, `Ctrl+Z`, `bg/fg`   | `jobs -l`, `fg %n`, `bg %n`, `disown`          | 셸 **내장**, 현재 세션 한정          |
| `kill`| 신호 전송/종료      | 정상 종료→불가 시 강제    | `-TERM(15)`, `-9(KILL)`, `-HUP`, `%n`          | **TERM 우선**, **KILL 최후의 수단**  |
